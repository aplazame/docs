# Configuración adicional en PrestaShop
Aquí encontrarás todo lo referente a la configuración adicional de nuestro plugin de Aplazame para PrestaShop.

## Botón de pago
Opciones de configuración para el botón o selector de pago con Aplazame.

Campo | Descripción
----- | -----------
Título del botón | Texto para el título a mostrar junto al botón de pago.
Descripción | Texto descriptivo del método de pago Aplazame.
Imagen del botón | URL de la imagen que quieras mostrar junto al botón de pago.
Selector del botón | Selector CSS para mostrar/ocultar el método de pago Aplazame.

## Widget
Opciones de configuración para el widget (tanto el [widget estándar](../../../api/widget/installment-payment#simulación) como el [widget v4](../../../api/widget/widget-v4)).

Campo | Descripción
----- | -----------
Activar widget legacy | Muestra el [widget estándar](../../../api/widget/installment-payment#simulación) en lugar del [widget v4](../../../api/widget/widget-v4).
Crear pedido al iniciar pago | Cada vez que un cliente inicia un proceso de financiación, se crea un pedido en el ecommerce. Se recomienda tener activa esta opción para evitar errores de expiración de checkouts.
CSS de precio variable | Selector CSS que indique el elemento que contiene el precio final del producto variable/carrito. Se actualiza en vivo si hay cambios en el precio.
Cuotas por defecto | Permite configurar el número de cuotas seleccionadas que mostrará el widget por defecto.
Hook del producto para el Widget | Permite seleccionar entre diferentes hooks dónde se va a mostrar el widget en la plantilla de producto.
Paga en 4 | Este campo provoca un cambio completo en la renderización del widget en caso de que el merchant soporte el producto **Pago en 4**. En caso de que no lo soporte o el total del checkout no cumpla los requisitos para el producto, se comportará exactamente igual que el [widget estándar](../../../api/widget/installment-payment#simulación) o que el [widget v4](../../../api/widget/widget-v4).
Pago inicial | Indica si debe mostrarse o no el importe del pago inicial en la parte superior del widget.
Texto legal | Indica si debe mostrarse el texto legal justo después del widget.
Widget si Aplazame no está disponible | Permite mostrar un widget diferente si el importe del producto (o carrito) está fuera de los límites configurados para tener Aplazame como método de pago. Con este widget, el cliente sabrá que aumentando el importe de la compra podrá financiarla (o pagarla en 4 pagos sin intereses).
:::tip
  Puedes editar el estilo del **Widget estándar** desde el [panel de control](https://vendors.aplazame.com/) en la sección **Preferencias > Ajustes del widget** pero ten en cuenta que, en los ajustes comunes con los del plugin, tienen prioridad los de este último.
:::

### Widget v4
Opciones de configuración exclusivas del **Widget v4**, el cual está activo siempre y cuando la opción `Widget legacy` (descrita en el apartado anterior) esté desactivada.

Campo | Descripción
----- | -----------
Alineación | Indica la alineación del widget. Se puede alinear a la `Izquierda`, a la `Derecha` o al `Centro`.
Color principal | Establece el color del fondo del botón y del borde.
Disposición | Se puede escoger la posición `Horizontal` (que cambiará automáticamente si no tiene el suficiente espacio) o `Vertical`.
Introducir cuota máxima | Cambia el widget de forma que el cliente deba introducir manualmente una cantidad máxima que desea pagar por cuota, calculando tras esto las opciones disponibles.
Marco | **Opción exclusiva del widget de producto.** Indica si debe mostrarse el borde de color.
:::tip
  El estilo del **Widget v4** sólo es editable desde la configuración del plugin, el estilo del **Widget estándar** configurable desde el [panel de control](https://vendors.aplazame.com/) no se tiene en cuenta.
:::
