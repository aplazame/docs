# Modo de pruebas
Aplazame proporciona un modo de pruebas para facilitar el proceso de integración antes de pasar a producción o para hacer cambios sin usar pedidos reales. Debe especificarse en la cabecera `Accept`, justo antes de la versión.

<Tabs>
  <TabItem value="http" label="http">

```http
  GET /orders HTTP/1.1
  Accept: application/vnd.aplazame.sandbox.v1+json
  Authorization: Bearer api_private_key
  Host: api.aplazame.com
```

  </TabItem>
  <TabItem value="shell" label="shell" default>

```powershell
  $ curl "https://api.aplazame.com/orders" \
  -H "Accept: application/vnd.aplazame.sandbox.v1+json" \
  -H "Authorization: Bearer api_private_key"
```

  </TabItem>
  <TabItem value="require" label="python" default>

```python
  import aplazame_sdk

  client = aplazame_sdk.Client('api_private_key', sandbox=True)

  response = client.orders()
```
  </TabItem>

  <TabItem value="php" label="php" default>

```php
  use Aplazame\Api\Client as AplazameClient;

  $aplazameClient = new AplazameClient("https://api.aplazame.com", AplazameClient::ENVIRONMENT_SANDBOX, "api_private_key");
  $result = $aplazameClient->get("/orders");
```
  </TabItem>
  <TabItem value="C" label="C#" default>

  ```csharp
  using Aplazame.Api;

  Client client = new Client("https://api.aplazame.com", Client.EnvironmentSandbox, "api_private_key");
  dynamic result = client.Get("/orders");
  ```
  </TabItem>
</Tabs>

Cabecera | Modo | Valor
-------- | ---------------- | -----
Accept | pruebas | application/vnd.aplazame.**sandbox**.v1+json
Accept | producción | application/vnd.aplazame.v1+json

:::tip
En la respuesta se indica el formato devuelto en la cabecera estándard `Content-Type` y la versión de la API en la cabecera `X-Aplazame-Media-Type`
:::

```http
HTTP/1.1 200 OK
Content-Type: application/json; charset=utf-8
X-Aplazame-Media-Type: aplazame.v1
```
