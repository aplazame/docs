# Migración a checkout v4
## Introducción

Esta página describe cómo migrar desde versiones anteriores de nuestro checkout a la versión más reciente del mismo, siendo el **checkout v4** la más actual hasta la fecha. La sección está orientada a todas aquellas integraciones que se hayan realizado vía API siguiendo los pasos de la [creación de checkout](..) antes de haber sido actualizado a la nueva versión.

:::caution
Si dispones de una integración vía plugin tendrás que dirigirte a la [sección correspondiente](../../../plugins/intro/) y actualizar el mismo siguiendo los pasos de descarga e instalación.
:::

## Migración

La migración puede ser muy sencilla o más laboriosa dependiendo del **proceso de confirmación** que se esté usando.

En caso de no estar usando el flujo de confirmación vía [notification_url](../#merchant), lo mejor es que se revise por completo el [proceso de checkout](../../checkout-process/installment-payment/#proceso-checkout), haciendo énfasis en la [creación de checkout](..) y [proceso de confirmación](../../checkout-confirmation), ya que estas primeras versiones de checkout varían parámetros y procesos básicos de las ultimas versiones.

En caso de ya estés usando el flujo de confirmación vía [notification_url](../#merchant), el cambio que debéis hacer para migrar al **checkout v4** es actualizar la cabecera `Accept` a la nueva versión.

Cabecera | Versión | Valor
-------- | ------- | -----
Accept | Actual (v4) | application/vnd.aplazame.**v4**+json
Accept | Obsoleta | application/vnd.aplazame.v1+json

:::tip
Para realizar pruebas en dicha versión, se dispone también de la variación **sandbox** disponible en la sección [modo de pruebas](../../test-environment/test-mode/).
:::
