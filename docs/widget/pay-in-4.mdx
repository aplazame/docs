## Pago en 4
En caso de que el merchant soporte el producto **'Pago en 4'**, se puede añadir el atributo `data-pay-in-4` al [widget de pago a plazos](#pago-a-plazos). Esto provocará un cambio en el diseño del widget, que mostará la información necesaria unicamente para 'Pago en 4'.

<div>
  <div
    data-aplazame-widget-instalments
    data-amount="8000"
    data-currency="EUR"
    data-country="ES"
    data-type="button"
    data-pay-in-4></div>
</div>

<aside>

  Si el widget tiene el atributo `data-pay-in-4` pero el merchant no soporta el producto 'Pago en 4' o la cantidad a financiar (`data-amount`) es demasiado alta, el widget se comportará exactamente igual que el [widget de pago a plazos](#pago-a-plazos), siempre que el merchant soporte el producto 'Pago a plazos', heredando todos sus atributos.
</aside>

``` html
<div data-aplazame-widget-instalments
     data-amount="8000"
     data-currency="EUR"
     data-country="ES"
     data-view="product"
     data-pay-in-4></div>
```

|data-pay-in | boolean | Sí | El atributo puede tener el valor **true** o estar vacío para que funcione correctamente. En caso de que no lo soporte o el total del checkout no cumpla los requisitos para el producto, se comportará exactamente igual que el [widget 'Pago a plazos'](#pago-a-plazos)
