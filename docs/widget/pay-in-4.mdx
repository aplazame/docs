# Pago en 4

## Integración
En caso de que el merchant soporte el producto **'Pago en 4'**, se puede añadir el atributo `data-pay-in-4` al [widget de pago a plazos](./installment-payment). Esto provocará un cambio en el diseño del widget, que mostará la información necesaria unicamente para 'Pago en 4'.



|<t id="table.parameter">Parámetro</t> | <t id="table.type">Tipo</t> | <t id="table.required">Requerido</t> | <t id="table.description">Descripción</t>|
|---------|---------|---------|---------|
|data-pay-in-4 | boolean | Sí | El atributo puede tener el valor **true** o estar vacío para que funcione correctamente. Este campo provoca un cambio completo en la renderización del widget en caso de que el merchant soporte el producto 'Pago en 4'. En caso de que no lo soporte o el total del checkout no cumpla los requisitos para el producto, se comportará exactamente igual que el [widget 'Pago a plazos'](./installment-payment)


:::caution
  Si el widget tiene el atributo `data-pay-in-4` pero el merchant no soporta el producto 'Pago en 4' o la cantidad a financiar (`data-amount`) es demasiado alta, el widget se comportará exactamente igual que el [widget de pago a plazos](./installment-payment), siempre que el merchant soporte el producto 'Pago a plazos', heredando todos sus atributos.
:::

### Simulación

<div className="widget-container" style={{ minHeight: '110px' }}>
  <div
    data-aplazame-widget-instalments
    data-amount="8000"
    data-currency="EUR"
    data-country="ES"
    data-type="button"
    data-pay-in-4></div>
</div>

``` html
<div data-aplazame-widget-instalments
     data-amount="8000"
     data-currency="EUR"
     data-country="ES"
     data-view="product"
     data-pay-in-4></div>
```
